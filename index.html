Mon programme d'apprentissage

  • Comprendre le fonctionnement d'un ordinateur et d'un réseau local

Un ordinateur (matériel) est composé principalement de :

  • Une carte mère (MotherBoard)
  • Un processeur (CPU)
  • De la mémoire vive (RAM)
  • Un disque dur (SSD)
  • Une carte gaphique (GPU)
  • Un BIOS (Basic Input Output System)
  • Une carte réseau (Ethernet, Wifi)
  • Des périphériques d'entrée/sortie (E/S) comme le clavier ou l'imprimante

  • Comprendre le système d'exploitation

Les deux systèmes d'exploitations (OS: Operating System) les plus utilisés en entreprise sont Linux et Windows Server.
Le système d'exploitation est une couche logicielle qui fait l'interface entre les applications et le matériel.
Le système d'exploitation est composé d'un noyau (kernel), de pilotes (drivers), d'outils système, de bibliothèques, de services.
Lorsque j'allume l'ordinateur:

  • Le BIOS s'allume en premier (il prépare le matériel: CPU, RAM, SSD GPU, ...)
  • Puis le BIOS lance le bootloader qui va charger le système d'exploitation

Mon stage

Stage Sentinel ArchiConsulting (1 mois)

Le projet que j’ai développé, avec l'aide de mon mentor, s’appelle « Météo Perso ». C’est une application qui surveille les ressources de l’ordinateur (processeur, mémoire, disque), enregistre les données dans une base PostgreSQL, puis les affiche sur une page web dynamique hébergée par un serveur web (python/Flask). Les informations sont présentées sous forme de graphiques. Enfin, l’accès à l’interface web est sécurisé grâce à un système d’authentification et au chiffrement TLS.

Semaine 1 (Langage C, SQL)

Objectifs
  • Lecture des informations système (CPU, RAM, disque) sous Linux
    • Fonction set_ram
      • Utilisation de /proc/meminfo pour lire la mémoire RAM totale et disponible
      • Calcul du % de RAM utilisé
    • Fonction set_cpu
      • Utilisation de /proc/cpuinfo pour récupérer le nombre de coeurs
      • Utilisation de la fonction getloadavg pour récupérer la charge moyenne
      • Calcul du % de CPU utilisé
    • Fonction set_disk
      • Utilisation de la fonction statvfs pour récupérer les stats du disque
      • Calcul du % de DISK utilisé
  • Stockage des mesures dans une base PostgreSQL
    • Utilisation de la bibliothèque PostgreSQL Libpq
    • Création d’une fonction d’insertion des métriques dans PostgreSQL
Outils
  • Langage C pour la collecte des métriques
  • PostgreSQL pour le stockage des données
  • Systèmes d'exploitation : Almalinux pour l'application en C et Rocky pour la base de donées PostgreSQL

Semaine 4 (Identification, chiffrement, Certificats, TLS)

Objectifs
  • Sécurisation de l'accès
    • PostgreSQL
      • Création d'une table users dans PostgreSQL
        • Insertion d'une colonne login et d'une colonne password
        • Isertion du mot de passe(hash bcrypt) dans la colonne password
    • Flask
      • Création d'une page /login avec un petit formulaire HTML(login.html)
        • Vérification du couple (login, mot de passe) côté Flask avec bcrypt
      • Utilisation de la session Flask pour 'retenir' que l'utilisateur est connecté (Cookie de session)
      • Redirection vers /graphics si l'authentification est réussi
      • Interdiction de l'accès à /graphics si la personne n'est pas connectée
      • Interdiction de l'accès à /data si la personne n'est pas connectée
    • Création d’un certificat racine (CA)
      • Génération d’un certificat d’autorité qui servira à signer les certificats du serveur
    • Création de la clé privée du serveur Flask
      • Génération du fichier server.key
    • Création de la requête de signature (CSR) du serveur Flask
      • Production du fichier server.csr, destiné à être signé par le CA
    • Création du fichier v3.ext
      • Ajout des informations nécessaires, dont le champ SAN (Subject Alternative Name)
    • Signature du certificat serveur
      • Génération du certificat final server.crt en utilisant :
        • la clé privée du serveur
        • le fichier server.csr
        • le certificat racine
    • Edge
      • Import du certificat racine dans les autorités de certification de confiance sur Windows 11
Outils
  • PostgreSQL pour la création de la table users
  • Système d'exploitation : Debian pour le serveur Flask en python
  • OpenSSL pour la génération des certificats
  • bcrypt pour le hachage des mots de passe
  • Edge pour l'import du certificat racine sur Windows 11
